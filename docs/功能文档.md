# Flovart 系统功能文档

> 版本：v0.2.0 · 技术栈：React 19 + TypeScript + Vite 6 + Tailwind CSS 4 + Zustand

---

## 一、项目概述

Flovart 是一个**开源 AI 图片/视频设计工具**（AI Canvas Studio），用户自带 API Key，在无限画布上完成从提示词生成、编辑到视频创作的全流程。所有数据存储在本地，不经过中间服务器。

---

## 二、画布引擎

### 2.1 无限画布
- 无限平移/缩放画布空间
- 支持多种**设计元素**：图片、视频、文字、形状、路径、箭头、分组
- 元素级变换：选中、拖拽移动、缩放、旋转
- 智能吸附对齐（网格对齐、元素对齐）
- 画布坐标系统与视口管理

### 2.2 设计工具
- **选择工具** — 单击/框选元素
- **画笔工具** — 自由绘制路径
- **形状工具** — 矩形、圆形等矢量形状
- **文字工具** — 画布文字输入与编辑
- **箭头工具** — 带箭头线段
- **裁切工具** — 图片裁切
- **遮罩工具** — 非破坏性图层蒙版，画笔擦除/恢复

### 2.3 图层管理
- 多图层堆叠与排序
- 图层可见性切换
- 图层面板（`LayerPanel`）

### 2.4 图片滤镜与调色
- 实时滤镜调节：亮度、对比度、饱和度、色调
- 模糊效果、复古滤镜等
- 滤镜面板（`ImageFilterPanel`）

### 2.5 多画板（Board）
- 多画板管理
- 每个画板独立维护撤销/重做历史

### 2.6 撤销/重做
- 基于 `historyState.ts` 的完整历史管理
- 画布元素操作的撤销/重做

### 2.7 A/B 对比
- 两张图片拖拽滑块进行前后对比
- 覆盖层组件（`ABCompareOverlay`）

## 三、AI 生成能力

### 3.1 文生图（Text-to-Image）
- 输入提示词生成图片
- 支持多 Provider：Google Gemini、OpenAI DALL-E、SDXL 等

### 3.2 图生图 / 图片编辑
- 选中图片 + 提示词 → 局部重绘（Inpaint）
- 选中图片 + 提示词 → 扩图（Outpaint），支持方向选择
- 背景移除（Remove Background）

### 3.3 文生视频（Text-to-Video）
- 提示词生成视频（支持 Veo / Sora 等）
- 多宽高比支持

### 3.4 批量生成
- 一次生成 2 张 / 4 张方案，供对比选择

### 3.5 提示词润色（Prompt Enhancement）
- 一键调用 LLM 自动优化提示词
- 支持多语言提示词增强

### 3.6 图片超分辨率（Upscale）
- AI 图片放大增强

## 四、AI Provider 系统

### 4.1 多 Provider 支持
12+ 模型服务商接入：

| Provider | 支持能力 |
|---|---|
| Google (Gemini) | 图片、视频、文本 |
| OpenAI | 图片、视频、文本 |
| Anthropic (Claude) | 文本 |
| DeepSeek | 文本 |
| Qwen (通义千问) | 图片、文本 |
| MiniMax | 图片、视频、文本 |
| 火山引擎 (Volcengine) | 图片、文本 |
| SiliconFlow | 图片 |
| Keling (可灵) | 图片、视频 |
| Flux | 图片 |
| Midjourney | 图片 |
| Banana | 图片、视频 |
| RunningHub | 图片、视频 |
| OpenRouter | 多模型聚合 |

### 4.2 BYOK（Bring Your Own Key）
- 用户自带 API Key，不经过中间服务器
- API Key 加密存储（`keyVault.ts` + `keyCodec.ts`）
- 凭据自动识别：粘贴 Key 自动识别 Provider + 拉取可用模型
- 批量 Key 管理

### 4.3 第三方服务适配
- 兼容 OpenAI-compatible 第三方端点（中转站、企业网关）
- 自动 Base URL 裁剪（`/v1/chat/completions` → `/v1`）
- Chat Completions 降级支持
- 多图片格式解析：`b64_json`、Data URL、HTTPS URL、Markdown 图片链接
- 能力声明：用户自定义 Provider 的能力配置

### 4.4 用量监控
- API 调用用量追踪（`usageMonitor.ts`）
- Provider 可用性测试

## 五、节点工作流引擎（Node Workflow）

### 5.1 可视化节点编辑器
- 基于有向图的视觉编程系统
- 节点拖拽、连线、布局
- 画布缩放/平移
- 节点选择、分组、对齐、分布
- 撤销/重做（80 层历史深度）
- 本地存储持久化

### 5.2 20 种节点类型
| 节点类别 | 节点类型 |
|---|---|
| **输入** | 提示词（Prompt）、加载图片、加载视频 |
| **生成** | 图片生成、视频生成、视频编辑 |
| **AI** | LLM 调用、提示词增强器 |
| **工具** | HTTP 请求、条件判断、Switch 分支、合并 |
| **模板** | 模板拼接 |
| **增强** | 超分辨率（Upscale）、人脸修复、背景移除 |
| **输出** | 保存到画布、保存到素材库 |
| **编排** | 预览节点 |
| **外部** | RunningHub 节点 |

### 5.3 节点连接与数据流
- 类型安全端口：文本 / 图片 / 视频 / JSON / 空值
- 端口兼容性校验
- Bezier 曲线连线
- 拓扑排序自动执行
- 节点输出级联传播

### 5.4 运行执行引擎（`workflowEngine.ts`）
- 拓扑排序确定执行顺序
- 顺序节点执行 + 数据传递
- 错误处理与重试
- 执行中止（Abort）支持
- 运行详情追踪

### 5.5 节点检视器
- 每类节点专属配置表单
- 字段类型：文本、多行文本、数字、下拉选择
- Provider/模型选择
- 节点输入预览

### 5.6 工作流模板
- 内置启动模板：《图生图》、《图生视频》流水线
- 短剧工作流模板（`makeShortDramaTemplate`）
- 模板分类（图片/视频/工具）

### 5.7 节点输出持久化
- 节点输出结果保存到 IndexedDB
- 支持图片 Data URL、视频 Blob、封面图

## 六、AI Agent 系统

### 6.1 Multi-Agent 群聊
- 多角色 Agent 协作（创意总监、提示词工程师、风格大师等）
- Agent 讨论后自动出图
- Agent 聊天面板（`AgentChatPanel`）

### 6.2 Agent Bridge
- 外部编程 Agent 通过 Flovart CLI 操作画布
- `window.__flovartAPI` 运行时接口
- AgentBridgePanel：状态/指令面板

### 6.3 Agent 编排 (`agentOrchestrator.ts`)
- 多 Agent 协作流程编排
- Action 注册与调度

## 十、Agent 接口工具（`tools/flovart/`）

### 10.1 Core 命令注册表（`core.js`）
19 个确定性命令：

| 命令 | 功能 |
|---|---|
| `help` | 帮助信息 |
| `setup` | 初始设置 |
| `status` | 系统状态 |
| `provider.status` | Provider 状态查询 |
| `provider.begin-setup` | 开始 Provider 配置 |
| `provider.select-model` | 选择模型 |
| `provider.test` | 测试 Provider 连接 |
| `canvas.list-media` | 列出画布媒体元素 |
| `canvas.add-image` | 添加图片到画布 |
| `canvas.add-video` | 添加视频到画布 |
| `canvas.clear-media` | 清空画布媒体 |
| `asset.list` | 列出素材库 |
| `generate.image` | 生成单张图片 |
| `generate.images-batch` | 批量生成图片 |
| `generate.video` | 生成视频 |
| `video.status` | 视频生成状态查询 |
| `export.project` | 导出项目 |

### 10.2 CLI（`cli.js`）
- 通过确定性命令操作 shadow runtime 或 file bridge
- provider-backed 命令写入 `.flovart/command-queue.json`，由浏览器 UI 轮询执行
- 用于 Codex、Claude Code、OpenCode、Shell 脚本

### 10.3 Runtime Client（`runtime-client.js`）
- 面向本地运行时的轻量客户端
- 不依赖 Chrome DevTools Protocol 端口
