图片 / 视频节点与渲染层怎么重构？

三种路线都保留 Flovart 自己的 ElementBar 与 PromptBar。请选择节点和大视频渲染的底层结构。

▶
DOM: image / video / bars SVG: edges / vector tools
推荐

A. 混合 DOM 媒体节点层

像参考项目一样用原生 img/video 节点并做视口裁剪；连线保留独立 SVG 层。改动集中，同时避开 foreignObject 和逐帧 Canvas 复制。

▶
SVG + foreignObject: everything

B. 继续使用 SVG foreignObject

只把 App.tsx 拆成组件，迁移最小；但大视频仍被 SVG 合成层约束，长期稳定性最弱。

▶
DOM: entire canvas interaction

C. 整个画布迁到 DOM 节点

最接近参考项目，但会波及路径、形状、裁剪、蒙版和现有选择交互，重写面最大，也最容易伤到你已有优势。

A 路线会保留

  • 现有 ElementBar
  • 现有 PromptBar 与 @ 引用
  • 图片/视频生成链路
  • 原 SVG 白板工具

A 路线会删除

  • 当前 App.tsx 内联端口和连线 JSX
  • 拖空白自动造 data-SVG 占位图
  • 视频进入 Konva 的强制路由
  • 点击细线立即删除的危险交互
